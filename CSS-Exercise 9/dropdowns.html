下拉選單 (Dropdowns)

使用 Bootstrap 下拉選單插件切換內文的 overlay 效果，顯示連結列表以及更多的內容。

概述

下拉選單是可切換的內文 overlay，用於顯示連結列表或其他內容。這些交互功能於 Bootstrap JavaScript 下拉選單插件提供。透過點擊切換，而不是滑入的方式，這是一個策劃好了的 設計決定。

下拉選單是透過第三方插件 Popper 所建構的，這提供了動態位置及 viewport 的監控。使用前請確認 popper.min.js 在 Bootstrap 的 JavaScript 之前，或者使用已經包含 Popper 的 bootstrap.bundle.min.js / bootstrap.bundle.js。因為不需要動態定位，Popper 並沒有用於導覽列下拉選單的定位。

親和性

WAI ARIA 該標準定義了 role="menu" widget，但這是專門用於應用程式的功能表，它們觸發動作或功能。 ARIA 選單只能包含功能表項目，核取方塊功能表項目，選項按鈕功能表項目，選項按鈕組和子功能表。

Bootstrap 的下拉選單是設計為通用的，適用於各種情況和標記結構。 例如，可以創建包含其他輸入和表單控制項（如搜索欄位或登錄表單）的下拉選單。 因此，Bootstrap 不希望（也不能自動添加）true ARIA 功能表所需的任何 role 和 aria- 屬性。 作者必須自己加入這些具體屬性。

然而，Bootstrap 為大多數標準鍵盤功能表操作加入內置支援，例如使用游標鍵移動單個 .dropdown-item 元素並使用 ESC 鍵關閉功能表的功能。

選單對齊

在預設的情況下，下拉選單會自動從父選單的頂部與左邊開始定位 100％。你可以使用定向 .drop* 類別更改此設定，但你也可以使用其他類別來控制他們。

在預設的情況下，下拉選單會自動從父選單的頂部與左邊開始定位 100％。將因此若將 .dropdown-menu-end 加入至 .dropdown-menu 則會使下拉選單靠右對齊。在 RTL 中使用 Bootstrap 時，方向則會相反，這意味著 .dropdown-menu-end 將出現在左側。

Heads up! Dropdowns are positioned thanks to Popper except when they are contained in a navbar.

響應式對齊

如果要使用響應式對齊，請增加 data-bs-display="static" 屬性來禁用動態定位，並使用響應式變數類別。

如果要讓在特定斷點以上讓下拉選單靠 右 對齊，請加上 .dropdown-menu{-sm|-md|-lg|-xl}-end。

如果要讓在特定斷點以上讓下拉選單靠 左 對齊，請加上 .dropdown-menu-end 和 .dropdown-menu{-sm|-md|-lg|-xl}-start.

請注意，您不需要在導覽列中的下拉按鈕加入 data-bs-display="static" 屬性，因為導覽列並沒有使用 Popper。

Mixins 用於生成基於 CSS 的符號，可以在 scss/mixins/_caret.scss 中找到。

                border-top: $caret-width solid;
                border-right: $caret-width solid transparent;
                border-bottom: 0;
                border-left: $caret-width solid transparent;
            }
            
            @mixin caret-up {
            border-top: 0;
            border-right: $caret-width solid transparent;
            border-bottom: $caret-width solid;
            border-left: $caret-width solid transparent;
            }
            
            @mixin caret-end {
            border-top: $caret-width solid transparent;
            border-right: 0;
            border-bottom: $caret-width solid transparent;
            border-left: $caret-width solid;
            }
            
            @mixin caret-start {
            border-top: $caret-width solid transparent;
            border-right: $caret-width solid;
            border-bottom: $caret-width solid transparent;
            }
            
            @mixin caret($direction: down) {
            @if $enable-caret {
                &::after {
                display: inline-block;
                margin-left: $caret-spacing;
                vertical-align: $caret-vertical-align;
                content: "";
                @if $direction == down {
                    @include caret-down();
                } @else if $direction == up {
                    @include caret-up();
                } @else if $direction == end {
                    @include caret-end();
                }
                }
            
                @if $direction == start {
                    &::after {
                        display: none;
                    }
            
                    &::before {
                        display: inline-block;
                        margin-right: $caret-spacing;
                        vertical-align: $caret-vertical-align;
                        content: "";
                        @include caret-start();
                    }
                }
            
                &:empty::after {
                    margin-left: 0;
                }
            }
            }

Usage

通過資料屬性或 JavaScript，下拉選單插件透過在父級的 .dropdown-menu 切換 .show 類別，藉以切換隱藏內容（下拉選單）。在關閉應用程式級別的下拉式功能表時，將依靠 data-bs-toggle="dropdown" 屬性，因此始終使用它是一個好方法。

在觸控功能的裝置上，打開一個下拉選單會加入一個空的 mouseover 事件處理給 <body> 元素的直接子元素。這怪異的手法是用來快速解決 iOS 的問題，否則在下拉選單之外的任何地方點擊會無法觸發關閉下拉選單的事件。一旦下拉選單被關閉，這些額外的 mouseover 事件就會被移除。

透過 JavaScript

透過 JavaScript 來調用下拉選單：

        
            var dropdownElementList = [].slice.call(document.querySelectorAll('.dropdown-toggle'))
            var dropdownList = dropdownElementList.map(function (dropdownToggleEl) {
                return new bootstrap.Dropdown(dropdownToggleEl)
            })
data-bs-toggle="dropdown" still required

無論是透過 JavaScript 調用您的下拉清單或使用 data-api， data-bs-toggle="dropdown" 都需要存在於下拉選單的觸發器元素中。

選項

可以透過資料屬性或 JavaScript 插件傳遞選項。對於資料屬性，將選項名稱加到 data-bs- 上，如 data-bs-offset=""。通過資料屬性傳遞選項時，請確保將選項名稱的大小寫類型從 camelCase 更改為kebab-case。 例如，不使用 data-bs-autoClose="false"而是使用 data-bs-auto-close="false"。

Name Type Default Description
boundary string | element 'clippingParents' 下拉選單的溢出限制邊界（僅適用於 Popper 的 preventOverflow modifier）。預設情況下，它是 'clippingParents' 並且可以接受 HTMLElement 引用（僅 JavaScript）。有關更多資訊，請參閱 Popper 的 detectOverflow 文件。
reference string | element | object 'toggle' 下拉選單的參考元素。可接受 'toggle'、'parent'、HTMLElement 引用或是提供 getBoundingClientRect 的物件的值。有關更多資訊，請參閱 Popper 的 constructor 文件 和 virtual element 文件。
display string 'dynamic' 預設的情況下，我們使用 Popper 來動態定位，而在 static 狀態下禁用
offset array | string | function [0, 2]

下拉選單相對於其目標的偏移量。你可以使用逗號分隔在資料屬性中傳遞字串，例如：data-bs-offset="10,20"

當一個函數被用來確定偏移量時，它會被一個包含 popper 位置、引用和 popper rects 的對象作為它的第一個參數調用。 觸發元素 DOM 節點作為第二個參數傳遞。該函數必須返回一個包含兩個數字的陣列： [skidding, distance]。

有關更多資訊，請參閱 Popper 的 offset 文件。

autoClose boolean | string true

配置下拉選單的自動關閉行為：

  • true - 下拉選單將通過點擊下拉選單內部或外部來關閉。
  • false - 下拉選單將通過點擊切換按鈕並手動調用 hide 或 toggle 方法來關閉。 (也不會被 esc 鍵關閉)
  • 'inside' - 下拉選單僅通過點擊下拉選單內部關閉。
  • 'outside' - 下拉選單僅通過點擊下拉選單外部關閉。
popperConfig null | object | function null

要改變 Bootstrap 預設 Popper config，請參閱 Popper's configuration。

當一個函數用於創建 Popper configuration 時，它會被一個包含 Bootstrap 的預設 Popper configuration 的物件調用。它可以幫助你使用預設配置並將其與你自己的配置合併。該函數必須為 Popper 返回一個 configuration object。

Using function with popperConfig

方法

Method Description
toggle 用於切換給定導覽列、分頁導覽的下拉選單。
show 用於顯示給定導覽列、分頁導覽的下拉選單。
hide 用於隱藏給定導覽列、分頁導覽的下拉選單。
update 更新一個元素的下拉選單定位。
dispose 銷毀一個元素的下拉選單。 (刪除 DOM 上儲存的元素)
getInstance 允許您獲取與 DOM 元素相關的 dropdown 實例的靜態方法，你可以使用像是：bootstrap.Dropdown.getInstance(element)
getOrCreateInstance 靜態方法，允許你獲取與 DOM 元素相關的 dropdown 實例，或者在未初始化的情況下創建一個新實例。 你可以使用像是：bootstrap.Dropdown.getOrCreateInstance(element)

事件

所有下拉事件都是在切換元素處被觸發，然後發生冒泡事件。因此您也可以在 .dropdown-menu 的父元素上添加事件監聽器。hide.bs.dropdown 和 hidden.bs.dropdown 事件具有 clickEvent 屬性 (僅當原始事件類型為 click) ，該屬性包含 click 的事件。

Method Description

shown.bs.dropdown 當這個物件可被看見時會觸發此事件 (完成 CSS 轉換後)。
hide.bs.dropdown 當 hide 實例方法被調用的時候立即觸發。
